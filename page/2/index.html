感受

  1. 谈谈幸福感

    现如今的中国，普通民众的幸福指数相对较低，而直接影响这个指数的是：我们需要工作多长时间才能换来属于自己的车子、房子。当前的中国社会，一个普通大学生毕业后到攒足自己房子首付的钱需要多少年，我们可以大体算一算。北上广深，其中的广州是房价最低的城市，郊区黄埔、萝岗、番禺、南沙等地的楼盘均价算1.5w一方，80平的小三房，其它凌杂的不算了，就算120w。三成首付36w。好，就拿这36w说说。普通的家庭，36w可是天文数字，如果你毕业后的目标就是这笔钱，那么三五年内，你的日子将会过的很不像样。再说下去就想哭了。
    幸福感真的低得快到马路边上了。。。

    不过，坦白来说，两个人一起生活、一起为我们的房子、车子、甚至是以后的孩子奋斗，相互体谅，可能这就是再平凡不过的幸福感了吧。。。

  2. 关于结婚

    如今毕业两年，业已到了成家的年纪（其实主要是女方再不结婚就已经严重晚婚晚育了）。关于结婚相关事宜比如求婚、婚纱照、酒席、婚礼等的花销，我简单算了一笔，得上到10w（当然，份子钱我没算进去）。嗯，先不说房子车子什么的，就连结个婚，感觉都很辛苦。但是，想想结婚是人生第一件大事，就算辛苦，咬咬牙也就过去了。

  2. 一哥们今天去下订思域10。不得不说的是，本田对于自家员工的购车补贴真是令人羡慕，每月1500的油费封顶补贴、1000的月供补助。嗯，关于买车的需求，还是应该好好衡量下了。但肯定是等到年后再考虑了，毕竟一个月3k的车贷，油费和停车费1.5k，加起来4.5k可不是小数目，尤其是现在差不多每月3.2k的房租水电的情况下，是万万不能想象的。。。

4. 构造并使用自定义风格的文本(1.26)

  • 问题

    你想要在不为每个属性创建单独的UI组件情况下能够在你的UI组件上显示丰富格式的文本。比如，你可能想要在一个UILabel里显示一个句子，这个句子里边有一个加粗的字。

  • 解决办法

    使用 NSAttributedString 或者其可变的版本－NSMutableAttributedString ，把它设置成UI组件的文本，比如说UILabel组件的attributedString属性，或者直接使用Attributed String的内置方法把文本绘制在画板上。

  • 讨论

    很多程序都会有这样的需求，要求能在一行文本里显示混合风格的字符串。比如第一个字是斜体，但第二个字是加粗，其它的是常规字体等。为了能实现这个效果，有些人必须使用webview ，但是，那并不是最优化的方案，因为 webview 在渲染字体的时候相当慢，这会严重影响你app的性能。iOS7里边，我们可以开始使用 属性字符串 。我不知道苹果在搞什么鬼，Mac开发早就能用上这个特性了，而iOS要在iOS7才能使用。

    构造属性字符串的最好方法就是：使用 NSMutableAttributedString 里的 initWithString: 方法，传一个NSString字符串进去。然后，对不同的部分设置不同的属性，我们使用 NSMutableAttributedString 中的 setAttributes:range: 方法。这个方法需要两个参数：

    • setAttributes：一个字典对象，键是字符属性，值取决于对应的键。以下是一些很重要的键：

      • NSFontAttributeName

        这个键的值是一个UIFont的对象，可以为指定范围的字符串定义这种字体。

      • NSForegroundColorAttributeName

        该键对应的值是UIColor对象。可以为指定范围的文本设置颜色。

      • NSBackgroundColorAttributeName

        可以为指定范围的文本设置背景颜色。

      • NSShadowAttributeName

        该键对应一个NSShadow对象，为指定范围的字符串定义阴影。

    • range：一个NSRange对象，定义了目标字符串的范围，包括起点，长度。

对于SB创建的VC，VC使用initWithCoder:进行初始化；

其他情况，VC是使用init:方法进行对象初始化，系统提供的UIViewController类init:实现过程中还会调用initWithNibName:bundle:方法完成VC的。

###VC.view的初始化 - loadView

无特殊情况，视图的初始化工作，可以在viewDidLoad里完成。

API-loadView的说明

You should never call this method directly. The view controller calls this method when its view property is requested but is currently nil. This method loads or creates a view and assigns it to the view property.

当VC访问view属性并发现其为nil时，就会去调用loadView加载视图，此方法正是为此而创建一个view并赋值给VC.view而生的。

If the view controller has an associated nib file, this method loads the view from the nib file. A view controller has an associated nib file if the nibName property returns a non-nil value, which occurs if the view controller was instantiated from a storyboard, if you explicitly assigned it a nib file using the initWithNibName:bundle: method, or if iOS finds a nib file in the app bundle with a name based on the view controller’s class name. If the view controller does not have an associated nib file, this method creates a plain UIView object instead.
如果VC已经跟一个nib文件关联，loadView方法会去nib文件里加载视图作为VC的view。一个VC的nibName属性非空就说明已经关联了一个nib文件，一般出现在使用StoryBoard设计VC、明确使用 initWithNibName:bundle: 去初始化VC、或者iOS在app里发现一个nib文件跟VC类同名。** 最后，如果VC没有关联任何的nib文件，那么loadView方法就会给VC.view创建一个空白的UIView对象。

If you use Interface Builder to create your views and initialize the view controller, you must not override this method.

You can override this method in order to create your views manually. If you choose to do so, assign the root view of your view hierarchy to the view property. The views you create should be unique instances and should not be shared with any other view controller object. Your custom implementation of this method should not call super.

If you want to perform any additional initialization of your views, do so in the viewDidLoad method.

如果你使用IB来创建你的视图并用来初始化你的VC，切勿重载此方法。

你可以通过重载此方法去手动创建你的视图。如果你这么干了，请把创建的视图的根视图赋值给VC.view。创建出来的视图应该是个独一无二的实例，且请勿与其他VC对象共享之。里边也不应该调用父类的这个方法。

如果你想对视图做一些额外的初始化工作，请在viewDidLoad方法里完成。

总之：无特殊情况，视图的初始化工作，可以在viewDidLoad里完成。

再见，viewDidUnload方法

从 iOS 6 开始，viewDidUnload 方法被废弃掉了，应用收到 memory warning 时也不会再调用 viewDidUnload 方法。

在 iOS6 时，当系统发出 MemoryWarning 时，系统会自动回收 bitmap 类。但是不回收 UIView 和 CALayer 类。这样即回收了大部分内存，又能在需要 bitmap 类时，通过调用 UIView 的 drawRect: 方法重建。如果贸贸然把self.view = nil;那么下次需要用到该视图时就需要重新做loadView的活了，而且还会触发viewDidLoad等等一系列创建视图的方法。

Use Lifetime Qualifiers to Avoid Strong Reference Cycles

You can use lifetime qualifiers to avoid strong reference cycles. For example, typically if you have a graph of objects arranged in a parent-child hierarchy and parents need to refer to their children and vice versa, then you make the parent-to-child relationship strong and the child-to-parent relationship weak. Other situations may be more subtle, particularly when they involve block objects.

大意译：你可以使用生命周期修饰符来避免保留环。比如，对于父子关系的对象，你可以让父对象强引用子对象，然后令子对象弱引用父对象。其他情况可能会比较难以察觉，特别是当他们涉及到块对象的时候。

In manual reference counting mode, block id x; has the effect of not retaining x. In ARC mode, block id x; defaults to retaining x (just like all other values). To get the manual reference counting mode behavior under ARC, you could use unsafe_unretained block id x;. As the name unsafe_unretained implies, however, having a non-retained variable is dangerous (because it can dangle) and is therefore discouraged.
