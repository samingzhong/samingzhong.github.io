关于Git与Tower的一些使用备忘

Git是目前最流行的版本管理工具，Git的功能相当强大，以git为核心的可视化工具有免费的Github的 Github Desktop、SourceTree以及收费Tower等。嗯我个人对比使用过之后，觉得还是Tower比较符合我自己。本文将会介绍下我在进行项目版本管理的时候比较重要的一些操作存做备忘。

本地回滚到某一个版本。

History，选中你希望回滚的版本，右键，Reset Head to “xxxx”。

说明：若要保留旧版本到当前版本的修改内容，记得勾上Keep Changes。否则修改内容应该是会被丢弃的。

关于文件忽略，根据使用情境会有所不同。

  • 文件/文件夹之前已经被添加进Git中管理了，现在想把它们忽略掉（这些文件的变化git不去做管理）。

    你可能需要动用git命令: git rm -r --cache _site/ 递归的把仓库根目录下的_site目录及其子目录的所有内容剔除git的管理，也就是以后这个文件还会在本地，但是服务器上的这会被删除。注意：除此之外，还需要把 _site/目录添加到 .gitignore 配置文件中，这样关于 _site目录的所有修改，git都不会监控。通常这种做法的目的是，不希望本地的数据被提交上到服务器上。

  • 其它一般的情况，直接配置.gitignore文件一般都能解决。

纠正上一次的提交日志。

Workingspace － Working Copy － 勾上 Amend Commit xxx选项，如果本次暂存区没有东西要提交，那么就是直接修改上次的提交日志。如果有，那么就是合并此次以及上次的提交，日志本次为准。

合并多个commit。

方法1：回退版本，保留变更内容，提交新版本、日志。此法简单粗暴，有些情况不大适合使用。

​
方法2：概括起来就是，r－重新编辑日志。f－向前合并commit，丢弃本次commit的日志。p－保留commit与日志，一般情况都能解决commit的合并需求。

  • 前提

    你的commits都还没有被push出去，你想根据功能合并多个commit。

  • 问题

    在我们把代码push出去之前，最好能整理好你的commit，否则可能影响日志的可读性。如何才能合并像上图的功能1.0和功能1.1成简洁的功能1开发呢（现实中，功能1中间常常会有N多个小commit，比如说实现了一个节点功能然后就commit了相应的代码，长期以往，一个完整的功能1，就有可能堆积了n多个commit，嗯，你也许会感觉很差）。

感受

  1. 谈谈幸福感

    现如今的中国，普通民众的幸福指数相对较低，而直接影响这个指数的是：我们需要工作多长时间才能换来属于自己的车子、房子。当前的中国社会，一个普通大学生毕业后到攒足自己房子首付的钱需要多少年，我们可以大体算一算。北上广深，其中的广州是房价最低的城市，郊区黄埔、萝岗、番禺、南沙等地的楼盘均价算1.5w一方，80平的小三房，其它凌杂的不算了，就算120w。三成首付36w。好，就拿这36w说说。普通的家庭，36w可是天文数字，如果你毕业后的目标就是这笔钱，那么三五年内，你的日子将会过的很不像样。再说下去就想哭了。

4. 构造并使用自定义风格的文本(1.26)

  • 问题

    你想要在不为每个属性创建单独的UI组件情况下能够在你的UI组件上显示丰富格式的文本。比如，你可能想要在一个UILabel里显示一个句子，这个句子里边有一个加粗的字。

  • 解决办法

    使用 NSAttributedString 或者其可变的版本－NSMutableAttributedString ，把它设置成UI组件的文本，比如说UILabel组件的attributedString属性，或者直接使用Attributed String的内置方法把文本绘制在画板上。

  • 讨论

    很多程序都会有这样的需求，要求能在一行文本里显示混合风格的字符串。比如第一个字是斜体，但第二个字是加粗，其它的是常规字体等。为了能实现这个效果，有些人必须使用webview ，但是，那并不是最优化的方案，因为 webview 在渲染字体的时候相当慢，这会严重影响你app的性能。iOS7里边，我们可以开始使用 属性字符串 。我不知道苹果在搞什么鬼，Mac开发早就能用上这个特性了，而iOS要在iOS7才能使用。

    构造属性字符串的最好方法就是：使用 NSMutableAttributedString 里的 initWithString: 方法，传一个NSString字符串进去。然后，对不同的部分设置不同的属性，我们使用 NSMutableAttributedString 中的 setAttributes:range: 方法。这个方法需要两个参数：

    • setAttributes：一个字典对象，键是字符属性，值取决于对应的键。以下是一些很重要的键：

      • NSFontAttributeName

        这个键的值是一个UIFont的对象，可以为指定范围的字符串定义这种字体。

      • NSForegroundColorAttributeName

        该键对应的值是UIColor对象。可以为指定范围的文本设置颜色。

      • NSBackgroundColorAttributeName

        可以为指定范围的文本设置背景颜色。

      • NSShadowAttributeName

        该键对应一个NSShadow对象，为指定范围的字符串定义阴影。

    • range：一个NSRange对象，定义了目标字符串的范围，包括起点，长度。
